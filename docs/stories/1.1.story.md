## Descrição

Como desenvolvedor, quero construir toda a UI do posgrad-tracker com dados hardcoded para validar o design e o fluxo de onboarding antes de conectar qualquer backend.

## Contexto

Esta é a primeira story. Não há banco de dados, autenticação real nem scraper ainda. O objetivo é ter a UI funcionando de ponta a ponta com dados falsos para validar:
- O fluxo de onboarding (6 telas)
- O design da tela de editais
- A experiência de "zero matches"

## Acceptance Criteria

- [ ] **AC1:** Projeto Next.js 15 criado com Tailwind CSS + shadcn/ui
- [ ] **AC2:** Arquivo `lib/mock-data.ts` com 10 editais mockados (áreas, estados, prazos variados)
- [ ] **AC3:** Tela 1 — Landing page com value prop e CTA "Começar agora"
- [ ] **AC4:** Tela 2 — Cadastro (formulário email/senha + botão Google — sem funcionar ainda)
- [ ] **AC5:** Tela 3 — Seleção de Grande Área (9 opções CAPES, mínimo 1 obrigatório, contador de selecionadas)
- [ ] **AC6:** Tela 4 — Seleção de Estados (27 UFs, checkbox EaD, botão habilitado após 1 seleção ou EaD marcado)
- [ ] **AC7:** Tela 5 — Aha moment com lista de editais mockados correspondentes à seleção
- [ ] **AC8:** Tela 6 — Tela de permissão de push (modal antes do browser pedir — sem funcionar ainda)
- [ ] **AC9:** Tela de Dashboard com lista de editais (filtro por área e estado)
