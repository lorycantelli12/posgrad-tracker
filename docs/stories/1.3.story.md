## Descrição

Como desenvolvedor, quero um script Python de matching que lê os JSONs do scraper, cruza com perfis de usuário definidos localmente e retorna uma lista ranqueada de editais relevantes — sem banco de dados ainda.

## Acceptance Criteria

- [ ] **AC1:** `matching/run.py` aceita parâmetros de linha de comando: `--grandes_areas`, `--estados`, `--niveis`, `--aceita_ead`
- [ ] **AC2:** Script lê `data/programas.json` e `data/editais_dou.json` + `data/editais_ies.json`
- [ ] **AC3:** Algoritmo de score implementado com pesos: área(40) + estado(30) + nível(20) + modalidade(10)
- [ ] **AC4:** Output é lista JSON ranqueada por score com campos: `edital_id`, `programa_nome`, `ies_nome`, `estado`, `nivel`, `prazo_inscricao`, `link_edital`, `score`
- [ ] **AC5:** Deduplicação: mesmo edital não aparece duas vezes no resultado
- [ ] **AC6:** Editais com prazo já vencido são excluídos do resultado
- [ ] **AC7:** Score zero = edital não aparece no resultado
- [ ] **AC8:** Testado com 5 perfis diferentes (arquivo `matching/test_profiles.json`)
- [ ] **AC9:** `matching/README.md` documentando uso e lógica dos pesos

## Scope

## Uso

```bash
# Perfil simples
python matching/run.py \
  --grandes_areas "ciencias_humanas" \
  --estados "SP,RJ" \
  --niveis "mestrado"

# Múltiplas áreas + EaD
python matching/run.py \
  --grandes_areas "engenharias,ciencias_exatas" \
  --estados "MG" \
  --niveis "mestrado,doutorado" \
  --aceita_ead

# Rodar todos os perfis de teste
python matching/run.py --test-all
```
