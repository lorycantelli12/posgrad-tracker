## Descrição

Como product owner, quero preparar o app para receber os primeiros usuários reais — com landing page definitiva, tratamento de erros, configurações de notificação e domínio customizado.

## Acceptance Criteria

### Landing Page
- [x] **AC1:** Landing page definitiva com value prop clara, sem referência a "dados mock"
- [x] **AC2:** Seção "Como funciona" com 3 passos (Configure → Monitoramos → Você é notificado)
- [x] **AC3:** CTA principal funcionando (leva ao cadastro real)

### UX e Estados
- [x] **AC4:** Todos os estados de erro tratados com mensagem amigável
- [x] **AC5:** Loading states nas telas que fazem queries
- [x] **AC6:** Estado vazio consistente em todas as listas
- [x] **AC7:** Página de preferências editáveis (`/preferencias`)

### Configurações de Notificação
- [x] **AC8:** Tela de configurações: pausar notificações (7/30/90 dias) — `/configuracoes`
- [x] **AC9:** Tela de configurações: alterar frequência do digest (diário/semanal/nunca)
- [x] **AC10:** Botão de cancelar conta com confirmação

### Performance e PWA
- [ ] **AC11:** Lighthouse score ≥ 80 — _(verificar após deploy)_
- [x] **AC12:** `public/manifest.json` configurado (já existia e está correto)
- [x] **AC13:** App instalável via browser (PWA) — manifest + themeColor no layout.tsx

### Domínio e Deploy
- [ ] **AC14:** Domínio customizado configurado na Vercel _(ação manual)_
- [ ] **AC15:** SSL ativo no domínio customizado _(automático na Vercel)_
- [ ] **AC16:** Vercel Analytics ativo _(ação manual — painel Vercel)_

### Alpha
- [ ] **AC17:** 5 usuários alpha convidados e cadastrados _(ação manual)_
- [ ] **AC18:** Feedback coletado de pelo menos 3 usuários alpha _(ação manual)_
- [ ] **AC19:** Bugs críticos reportados pelo alpha corrigidos _(após alpha)_

## Scope

**IN:**
- Landing page final
- Tratamento de erros
- Configurações de notificação
- PWA
- Domínio customizado
- Alpha com 5 usuários

**OUT:**
- App nativo iOS/Android

- Dashboard de analytics para o admin
