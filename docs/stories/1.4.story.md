## Descrição

Como desenvolvedor, quero integrar o OneSignal ao app Next.js e validar que push notifications chegam no browser com o conteúdo correto — sem banco de dados real ainda (usar user_id hardcoded para testes).

## Acceptance Criteria

- [ ] **AC1:** Conta OneSignal criada, App ID e API Key configurados _(ação manual — criar em onesignal.com)_
- [x] **AC2:** `react-onesignal` instalado e inicializado no `app/layout.tsx`
- [x] **AC3:** Modal de permissão de push funcionando (tela customizada antes do browser perguntar)
- [ ] **AC4:** Push notification chegando no Chrome (desktop) após aceitar permissão _(requer credenciais AC1)_
- [ ] **AC5:** Push notification chegando no Safari mobile (iOS) — documentado em `notify/README.md` _(requer credenciais AC1)_
- [x] **AC6:** Conteúdo do push correto: título com nome do programa, corpo com IES + estado + prazo
- [x] **AC7:** Clique no push abre URL correta do edital (deep link `/editais/{id}`)
- [x] **AC8:** `notify/send_push.py` — script Python que dispara push via API OneSignal para `external_id` hardcoded
- [x] **AC9:** Testado com 3 templates diferentes (match novo, deadline 7 dias, digest)
- [x] **AC10:** `.env.local.example` criado com `NEXT_PUBLIC_ONESIGNAL_APP_ID` + `ONESIGNAL_API_KEY`
- [x] **AC11:** `notify/README.md` documentando como testar push manualmente

## Templates de Push (referência)

```
Template 1 — Match Novo:
Título: "🎓 {programa_nome}"

URL:    "/editais/{edital_id}"

Template 2 — Deadline 7 dias:
Título: "⏰ Prazo se encerrando"
Corpo:  "{programa_nome} — inscrições encerram em 7 dias"

Template 3 — Digest:
```
