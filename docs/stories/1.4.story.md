# Story 1.4 — Notificações com OneSignal

**Epic:** EPIC-001 PosGrad Tracker
**Story ID:** 1.4
**Status:** Draft

**Estimativa:** 2 dias
**Depende de:** 1.1 (UI pronta), 1.3 (matching validado)

## Descrição

Como desenvolvedor, quero integrar o OneSignal ao app Next.js e validar que push notifications chegam no browser com o conteúdo correto — sem banco de dados real ainda (usar user_id hardcoded para testes).

## Acceptance Criteria

- [ ] **AC1:** Conta OneSignal criada, App ID e API Key configurados
- [ ] **AC2:** `react-onesignal` instalado e inicializado no `app/layout.tsx`
- [ ] **AC3:** Modal de permissão de push funcionando (tela customizada antes do browser perguntar)
- [ ] **AC4:** Push notification chegando no Chrome (desktop) após aceitar permissão
- [ ] **AC5:** Push notification chegando no Safari mobile (iOS) — ou documentado por que não funciona ainda
- [ ] **AC6:** Conteúdo do push correto: título com nome do programa, corpo com IES + estado + prazo
- [ ] **AC7:** Clique no push abre URL correta do edital (deep link `/editais/{id}`)
- [ ] **AC8:** `notify/send_push.py` — script Python que dispara push via API OneSignal para `external_id` hardcoded
- [ ] **AC9:** Testado com 3 templates diferentes (match novo, deadline 7 dias, digest)
- [ ] **AC10:** Variáveis de ambiente configuradas: `NEXT_PUBLIC_ONESIGNAL_APP_ID` + `ONESIGNAL_API_KEY`
- [ ] **AC11:** `notify/README.md` documentando como testar push manualmente

## Scope

**IN:**
- OneSignal integrado ao Next.js
- Push chegando no browser
- Script Python de disparo manual
- 3 templates de push

**OUT:**
- Push automático via Edge Function (Story 1.5)
- Emails Resend (Story 1.5)
- Deduplicação real (Story 1.5)
- Limite diário real (Story 1.5)
