## Descrição

Como operador do produto, quero configurar todas as credenciais de produção (Supabase, OneSignal, Resend, GitHub Actions cron) para que o app saia do modo mock e comece a notificar usuários reais.

## Contexto

Toda a infraestrutura foi codificada no Epic 1 (Stories 1.4 e 1.5) mas **nenhuma credencial real foi configurada**. O app ainda usa dados mock no frontend e os scrapers salvam apenas em JSON local. Esta story conecta os pontos.

**O que precisa de credenciais:**
- **Supabase** — banco live com schema (migrations já criadas em `supabase/migrations/`)
- **OneSignal** — push notifications (código em `notify/send_push.py` e `app/onboarding/notificacoes/`)
- **Resend** — emails transacionais (código em `notify/`)
- **GitHub Actions** — cron diário para rodar scrapers + matching + notify

## Acceptance Criteria

- [ ] **AC1:** Supabase projeto criado (ou existente), URL e ANON_KEY configurados no Vercel como env vars

- [ ] **AC8:** GitHub Secrets configurados: `SUPABASE_URL`, `SUPABASE_SERVICE_KEY`, `ONESIGNAL_REST_API_KEY`, `RESEND_API_KEY`

## Notas de Dev

- Supabase free tier: 500MB, 50k MAU — suficiente para alpha
- OneSignal free tier: unlimited push — OK para < 10k subscribers
- Resend free tier: 3k emails/mês — OK para alpha
- GitHub Actions free: 2000 min/mês — cron diário (~5min/run) = ~150min/mês, dentro do free
- Service Worker do OneSignal precisa estar em `/public/` — verificar se já existe
- **Ordem de configuração:** Supabase primeiro (frontend depende) → OneSignal → Resend → Actions
